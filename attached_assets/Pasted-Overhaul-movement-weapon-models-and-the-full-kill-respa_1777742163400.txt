SECTION 1 — REALISTIC FPS MOVEMENT
─────────────────────────────────────────

Rewrite the movement system in Controls.js completely. 
The current movement feels floaty and instant. Replace it with this:

SPEED VALUES:
  walkSpeed   = 5.5  units/sec
  sprintSpeed = 8.5  units/sec  (hold Shift)
  crouchSpeed = 2.8  units/sec  (hold C or Ctrl)
  airSpeed    = 3.5  units/sec  (reduced control while airborne)

ACCELERATION & FRICTION:
  Do NOT set velocity directly from input. Instead use acceleration/deceleration:
  - groundAccel  = 60  (fast snap to speed when key held — responsive but not instant)
  - groundFrict  = 50  (fast stop when key released — no sliding)
  - airAccel     = 18  (sluggish mid-air direction change — realistic)
  - airFrict     = 4   (minimal air friction — momentum carries)

  Each frame:
    wishDir = normalized input direction (from WASD in camera-space)
    if grounded:
      velocity.x = lerp toward (wishDir.x * currentSpeed) at groundAccel * delta
      velocity.z = lerp toward (wishDir.z * currentSpeed) at groundAccel * delta
      apply groundFrict when no input: velocity.xz *= max(0, 1 - groundFrict * delta)
    else (airborne):
      velocity.x += wishDir.x * airAccel * delta
      velocity.z += wishDir.z * airAccel * delta
      clamp horizontal speed to airSpeed
      apply airFrict: velocity.xz *= max(0, 1 - airFrict * delta)

GRAVITY & JUMPING:
  gravity     = -22  units/sec²
  jumpForce   = 7.5  units/sec  (gives about 1.3 unit jump height — realistic)
  eyeHeight   = 1.65 units (camera Y when standing)
  crouchHeight= 1.1  units (camera Y when crouched)

  Gravity applied every frame: velocity.y += gravity * delta
  Ground check: raycast straight down 0.05 units below player feet.
    If hit: set grounded=true, velocity.y = max(0, velocity.y)
  Jump: only if grounded, set velocity.y = jumpForce, grounded=false

  Smooth crouch: lerp camera Y between eyeHeight and crouchHeight at speed 10/s

MOUSE LOOK:
  Horizontal sensitivity = 0.0018  (multiply mouseDelta.x by this for yaw)
  Vertical sensitivity   = 0.0018
  Pitch clamp: -80 to +80 degrees
  Apply yaw to the player's Y rotation (whole body turns)
  Apply pitch to camera only (head tilts, body stays upright)
  Store mouseSensitivity in localStorage, default 0.0018

HEAD BOB:
  When grounded and moving, apply a subtle head bob to camera Y:
    bobSpeed = 10 (walk) or 14 (sprint)
    bobAmount = 0.028 (walk) or 0.042 (sprint)
    camera.position.y += sin(bobTimer * bobSpeed) * bobAmount
    bobTimer += delta when moving, reset toward 0 when stopped
  When airborne or standing still: smoothly damp bob back to 0
  This makes walking feel physical without being nauseating

EMIT movement to server every 50ms (not every frame):
  Send { x, y, z, rotY, crouching }

SECTION 4 — CLASS-SPECIFIC WEAPON SOUNDS
─────────────────────────────────────────

Rewrite SoundSystem.js to have per-class gunshot sounds using Web Audio API only.

playGunshot(weaponType):  accepts 'ak47', 'smg', or 'sniper'

'ak47' — full rifle crack, mid-low frequency, sharp transient:
  a) Crack tone: OscillatorNode square, 160hz → 80hz sweep over 90ms, gain 0.7 → 0
  b) Body noise: white noise buffer 0.2s, BandpassFilter center 400hz Q=1.5, gain 0.5 → 0 over 180ms
  c) Sub punch: OscillatorNode sine, 80hz → 30hz over 70ms, gain 0.6 → 0
  d) Reverb tail: white noise buffer 0.4s, LowpassFilter 600hz, gain 0.15 → 0 over 400ms
  Total character: punchy, authoritative, a bit bassy

'smg' — fast light crack, higher pitch, very short:
  a) Crack: OscillatorNode square, 280hz → 140hz over 55ms, gain 0.5 → 0
  b) Noise: white noise 0.1s, HighpassFilter 500hz, gain 0.4 → 0 over 90ms
  c) No sub. No reverb tail.
  Total character: snappy, quick, light

'sniper' — massive loud crack with long echo tail:
  a) Initial crack: OscillatorNode sawtooth, 120hz → 40hz over 40ms, gain 0.9 → 0
  b) Pressure wave: white noise 0.08s, LowpassFilter 200hz, gain 0.8 → 0 over 60ms
  c) Echo tail: white noise 1.2s, BandpassFilter 300hz Q=0.5, gain 0.3 → 0 over 1200ms (long decay)
  d) Sub boom: OscillatorNode sine, 50hz → 20hz over 100ms, gain 0.7 → 0
  Total character: thunderous, echoey, intimidating

playReload(weaponType):
  'ak47'  — two mechanical clicks: 2 x short noise burst (30ms, 0.3 gain) 400ms apart
  'smg'   — magazine snap: one sharp click (20ms) + soft plastic thud (40ms) 200ms later
  'sniper'— bolt action: clunk (60ms low noise) + slide (80ms filtered noise) + clunk (60ms) over 600ms

All sounds: call audio.resume() on first user click (browser autoplay policy).

On 'player:shoot' received:

            * Set target.alive = false
            * Emit 'player:killed' to ALL: { killerId, killerName, victimId, victimName, killerClass }
